Your Net IDE Package project has been successfully generated.

Your project is now configured to be compiled and started as a Net IDE Package. However, one piece of configuration is still missing. If this is a Core package for a new Net IDE application, the context for this package still needs to be registered. In a production environment, this registration would be done through the setup, but for a development environment, this must be done by hand.

These steps only apply if this is a Core package. For plugins to an existing application, you need to have the application installed, which already ensures that the context has been registered.

  • Locate the full output path of your project. This is the bin\Debug directory in your project.

  • Open the Registry Editor.

  • Locate the path HKEY_CURRENT_USER\Software\Net IDE. If it does not exist yet, create it.

  • Under the HKEY_CURRENT_USER\Software\Net IDE key, create a new key with the name of your context.

  • Under the context key, create a new String Value with the name InstallationPath.

  • Change the value of the InstallationPath String Value to the full output path of your project.
